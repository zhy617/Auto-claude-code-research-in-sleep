# 多模态基础模型：机制拆解（输入到训练）

**日期**：2026-03-24  
**主题**：多模态信息如何进入 LLM、token 如何组织、对齐与训练阶段如何设计  
**使用说明**：本文面向 base/foundation 机制分析；区分文献已验证结论与推断，并标注置信度。

---

## 1. 端到端流程（框架性，文献已验证 + 归纳）

| 阶段 | 内容 | 典型实现形态 |
|------|------|----------------|
| 输入表征 | 图像块/帧/频谱/文本串 | Patchify、重采样、分词 |
| 模态编码 | 视觉塔、音频编码器、文本嵌入 | ViT、CLIP/SigLIP、Whisper 类、字/子词嵌入 |
| 连接层 | 维度与语义对齐 | projector、Q-Former、Perceiver Resampler、cross-attn 模块 |
| Token 注入 | 与文本序列拼接或并行注意力 | 前缀视觉 token、交错占位、专用注意力 |
| 训练 | 多阶段、多目标 | 对齐预训练 +（可选）指令微调；本 survey 重「对齐与预训练」 |
| 推理 | 自回归或生成 | 与 LLM 相同的解码逻辑扩展至多模态前缀 |

**推断**：多数工业方案在「连接层」与「token 预算」上迭代最频繁。  
**置信度：高**。

## 2. 核心问题回答

### 2.1 多模态信息如何进入 LLM？

**文献已验证**：路径包括——(1) **连续嵌入**经 projector/Q-Former/resampler 映射到 LLM 输入空间；(2) **交叉注意力**从视觉侧读入冻结或半冻结特征；(3) **离散统一 token** 与文本共享自回归头（Chameleon/Emu3 等路线）。

**推断**：不存在单一最优入口；选择与「是否保留 LLM 权重」「token 预算」「延迟」强相关。  
**置信度：高**（路径多样性）；**中**（闭源细节）。

### 2.2 视觉/音频 token 与文本如何共存？

| 组织方式 | 机制要点 | 代表线索 |
|----------|----------|----------|
| 前缀拼接 + 全序列自注意力 | 视觉 token 作为前缀，与文本一起 self-attn | LLaVA 类常见描述 |
| 交叉注意力分支 | 文本隐状态 query 视觉 memory | Flamingo 类 |
| 统一离散序列 | 模态皆为 token ID | Kosmos、Chameleon、Emu3 |
| 时间维堆叠/池化后再前缀 | 控长、适配 KV | 视频/音频扩展 |

**文献已验证**：「拼接自注意力 vs 额外交叉注意力」是公开论文中反复出现的二分法。

### 2.3 对齐依赖什么？

| 机制 | 作用 | 代价/瓶颈线索 |
|------|------|----------------|
| Projector（线性/MLP） | 低复杂度对齐 | 容量不足时信息瓶颈 |
| Q-Former | 可学习 query 压缩视觉 | 查询数量与语义保留折中 |
| Cross-attention | 强表达力 | 参数与延迟、与冻结策略耦合 |
| 统一自回归 | 端到端模态建模 | 数据与词表设计难度大 |

**推断**：「对齐」往往是多阶段目标（对比、caption、自回归）叠加的结果，非单一损失可解释。  
**置信度：中**。

### 2.4 训练通常分几阶段？为什么？

**文献已验证**（归纳主流公开配方）：常见包括 **(1) 视觉-语言对齐**（冻结或部分冻结编码器/LLM）、**(2) 联合或指令式微调**；部分工作另有 **纯视觉或纯语言预训练** 作为前置。  
**原因（推断）**：降低灾难性遗忘、稳定优化、分阶段消化数据分布。**置信度：高**（分阶段普遍存在）；**中**（最优阶段数因模型而异）。

## 3. 关键设计变量（≥10）与机制含义

| # | 变量 | 文献已验证的选项示例 | 为何影响全局 |
|---|------|----------------------|--------------|
| 1 | 编码器选型 | CLIP / SigLIP / DINO / EVA / 语音编码器 | 决定视觉/音频语义与细粒度上限 |
| 2 | Connector 复杂度 | linear、MLP、Q-Former、resampler、cross-attn | 决定压缩与可塑性的折中 |
| 3 | LLM 冻结策略 | 全冻结、逐层解冻、LoRA、全参微调 | 影响对齐自由度与遗忘风险 |
| 4 | 视觉 token 预算与分辨率 | 固定 grid、动态分辨率、多尺度 | 直接决定上下文与 KV 压力 |
| 5 | 位置编码与序列组织 | 1D 展开、M-RoPE、时间戳式扩展 | 影响多图/多帧/文档结构推理 |
| 6 | 训练目标组合 | caption、对比、自回归、指令 | 决定「对齐 vs 对话能力」配比 |
| 7 | 注意力形态 | 纯拼接 self-attn、额外 cross-attn | 影响实现复杂度与信息流 |
| 8 | 数据配比与 curriculum | 先对齐后指令、混合比例 | 影响泛化与专项能力 |
| 9 | 音频/视频时间压缩 | 帧采样、时序池化、轻量时序编码 | 控长与保真的矛盾 |
| 10 | 离散统一 token vs 连续嵌入 | VQ/VAE token vs projector | 统一建模 vs 训练稳定性 |

## 4.「为何有效」与「为何成瓶颈」（对照）

| 设计 | 为什么能工作（机制级） | 为什么易成后续瓶颈 |
|------|------------------------|---------------------|
| 冻结强编码器 | 保留视觉表征质量、减算力 | 与 LLM 能力域错配 |
| 轻量 projector | 快速对齐、易复现 | 细粒度与长链推理信息不足 |
| Q-Former / resampler | 显式压缩视觉到固定长度 | query 数限制「可检索细节」 |
| Cross-attention 注入 | 灵活读写视觉 memory | 延迟、显存、实现与训练稳定性 |
| 动态分辨率 / 多图 | 提高有效像素覆盖 | token 膨胀、位置编码与推理成本 |
| 统一 token | 模态边界模糊、端到端 | 数据与词表、训练难度高 |

**推断**：瓶颈往往在「接口容量」与「序列长度」交汇处爆发。  
**置信度：中**。

---

## 5. 仍困难的问题（归纳）

**文献已验证**：公开讨论中反复提到 grounding、长视频时序推理、跨模态组合推理的数据与评测局限。  
**推断**：根因包括 **(1) 像素级 grounding 与语言抽象不对齐**、**(2) 时间维压缩丢因果**、**(3) 多模态联合分布覆盖不足**。  
**置信度：中**（方向共识高，量化因果难）。

---

## 6. 小结

- **文献已验证**：多模态进入 LLM 的路径可归纳为连续对齐、交叉注意力、统一 token 三大类；训练多分阶段。  
- **推断**：后续优化将集中在「connector—token 预算—位置/时间编码」三角。  
- **置信度：高**（框架）；**中**（个案最优配置）。
