# 多模态 Base 模型：潜在优化入口（方向归纳）

**日期**：2026-03-24  
**主题**：从模块出发的优化方向盘点；**不**提出具体论文 idea，仅归纳研究空间  
**使用说明**：每节含「文献已验证」的主流做法与「推断」的失败模式；文末给出排序与深读建议。

---

## 1. 十个潜在优化入口（机制级）

### 入口 1：Connector 容量与压缩

| 维度 | 内容 |
|------|------|
| **对应模块** | projector、Q-Former、Perceiver Resampler、cross-attn bridge |
| **为何重要** | 决定视觉/音频到 LLM 的可用信息通量 |
| **当前主流** | 线性/MLP、固定 query 数、浅层对齐预训练 |
| **常见失败模式** | 细节丢失、指代错误、长链推理断在接口处 |
| **高质量研究问题？** | **是**（可分离变量多、可对照强） |

**文献已验证**：连接器设计在 BLIP-2、LLaVA、Flamingo 等中有明确描述。  
**置信度：高**。

---

### 入口 2：动态分辨率与 Token Allocation

| 维度 | 内容 |
|------|------|
| **对应模块** | patch 选择、多尺度融合、全局/局部二路径 |
| **为何重要** | 在固定 token 预算下最大化有效像素覆盖 |
| **当前主流** | 动态分辨率、分块、任意分辨率展开 |
| **常见失败模式** | 块边界 artifact、计算抖动、实现复杂度高 |
| **高质量研究问题？** | **是**（与系统瓶颈强相关） |

**文献已验证**：Qwen2-VL、LLaVA-NeXT 等公开论述。**置信度：高**。

---

### 入口 3：视觉 Token Pruning 与保真

| 维度 | 内容 |
|------|------|
| **对应模块** | 视觉 token 选择、重要性估计、与 LLM 联合剪枝 |
| **为何重要** | 直接缓解 KV 与延迟 |
| **当前主流** | 池化、固定 stride、启发式降采样 |
| **常见失败模式** | 剪错关键 token，小物体/字丢失 |
| **高质量研究问题？** | **是**（可定义细粒度评价） |

**推断**：工业界方案多于公开论文。**置信度：中**。

---

### 入口 4：长视频时间压缩

| 维度 | 内容 |
|------|------|
| **对应模块** | 帧采样、时序 encoder、事件级 token |
| **为何重要** | 视频长度与注意力成本矛盾尖锐 |
| **当前主流** | 均匀采样、轻量 3D patch、与文本交错 |
| **常见失败模式** | 因果顺序混淆、快动作丢失 |
| **高质量研究问题？** | **是**（数据与架构耦合深） |

**文献已验证**：Video-LLaVA 等管线有公开讨论。**置信度：高**（问题存在）；**中**（最优解）。

---

### 入口 5：Grounding-aware 表征

| 维度 | 内容 |
|------|------|
| **对应模块** | 编码器中层特征、connector、可选 grounding 头 |
| **为何重要** | 缓解语义—像素鸿沟 |
| **当前主流** | 高分辨率输入、专用数据混合 |
| **常见失败模式** | 通用 loss 下 grounding 仍弱 |
| **高质量研究问题？** | **是**（任务定义清晰） |

**文献已验证**：grounding 难点在评测文献中反复出现。**置信度：高**。

---

### 入口 6：视觉编码器与 LLM 能力匹配

| 维度 | 内容 |
|------|------|
| **对应模块** | 预训练 ViT/SigLIP、解冻策略 |
| **为何重要** | 错配导致「看得清但说不清」或反之 |
| **当前主流** | 冻结塔 + 轻量对齐；部分工作解冻顶层 |
| **常见失败模式** | 解冻引发不稳定或遗忘 |
| **高质量研究问题？** | **是**（需控制变量实验） |

**文献已验证**：换塔与解冻在多篇技术报告中有消融。**置信度：高**。

---

### 入口 7：位置编码与 M-RoPE 类扩展

| 维度 | 内容 |
|------|------|
| **对应模块** | RoPE 变体、多维位置、文档布局 |
| **为何重要** | 多图、多帧、结构化输入依赖位置可组合性 |
| **当前主流** | 1D 展开 + 特殊位置标记；部分模型引入多维 RoPE |
| **常见失败模式** | 布局错乱、跨页指代失败 |
| **高质量研究问题？** | **是**（可形式化几何/布局子集） |

**文献已验证**：M-RoPE 等在官方材料中有模块名。**置信度：高**（存在性）；**中**（普适优势）。

---

### 入口 8：多图/文档组织

| 维度 | 内容 |
|------|------|
| **对应模块** | 分隔符、页级 token、阅读顺序 |
| **为何重要** | 真实场景多为多页 PDF、多截图 |
| **当前主流** | 特殊 token 分隔、顺序拼接 |
| **常见失败模式** | 长文档注意力稀释、跨图推理弱 |
| **高质量研究问题？** | **是**（与长上下文研究交汇） |

**推断**：产品需求驱动，公开 base 论文略分散。**置信度：中**。

---

### 入口 9：模态平衡训练

| 维度 | 内容 |
|------|------|
| **对应模块** | 数据管线、损失权重、采样器 |
| **为何重要** | 缓解 modality imbalance |
| **当前主流** | 分阶段、混合比例、重采样 |
| **常见失败模式** | 某一模态「吃掉」梯度 |
| **高质量研究问题？** | **条件性**（需严谨监控与消融） |

**文献已验证**：多阶段训练普遍性。**置信度：高**（现象）；**中**（最优策略）。

---

### 入口 10：原生 Unified Token

| 维度 | 内容 |
|------|------|
| **对应模块** | 词表、tokenizer、离散视觉 codebook |
| **为何重要** | 统一自回归可能简化接口与推理路径 |
| **当前主流** | VQ 视觉 token + 文本 BPE；部分模型统一训练 |
| **常见失败模式** | 训练不稳、重建与语义目标冲突 |
| **高质量研究问题？** | **是**（探索空间大、风险也高） |

**文献已验证**：Chameleon、Emu3 等路线。**置信度：高**。

## 3. 最值得继续深入的三方向（论文/源码）

1. **动态分辨率 + token allocation + 长上下文**：与 Qwen2-VL、LLaVA-NeXT、系统侧 KV 文献交叉阅读。  
2. **Connector 家族（Q-Former / projector / cross-attn）对照**：从 BLIP-2、Flamingo、LLaVA 三条线平行读。  
3. **统一 token 路线（Chameleon / Emu3）与连续嵌入路线**：对比训练稳定性与模态扩展路径。

**推断**：三者覆盖「序列长度—接口—建模范式」主轴。**置信度：中**。
